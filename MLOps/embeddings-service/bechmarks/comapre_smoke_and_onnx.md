# Сравнение `smoke_run.csv` vs `run_2026_03_08_onnx.csv`

Сравнивались файлы:

- `benchmark/results/smoke_run.csv`
- `benchmark/results/run_2026_03_08_onnx.csv`

Формула изменения: `onnx - smoke` (в скобках процент относительно `smoke`).

## Ключевые наблюдения

- Для маленьких батчей (`single_request`, `concurrent_load`, `batch=1`) ONNX заметно быстрее: latency ниже на 53-85%, throughput выше на 127-518%.
- На среднем батче (`batch=8`) ONNX немного лучше по throughput (+14.5%) при сопоставимой latency.
- На больших батчах (`batch=16`, `batch=32`) ONNX проигрывает: p95/p99 существенно выше, `texts/s` ниже на 10.3% и 36.6% соответственно.
- Память (`max_rss_mb`) у ONNX стабильно выше во всех сценариях: примерно +32-39%.

## Итог

ONNX-конфигурация выгодна для low-batch/latency-sensitive нагрузки, но при крупных батчах текущая реализация показывает деградацию по хвостовой latency и throughput при большем потреблении памяти.
